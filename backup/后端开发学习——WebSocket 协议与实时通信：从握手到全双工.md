<p class=MsoNormal>引言<span lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal>在现代<span lang=EN-US> Web </span>开发中，实时通信已成为许多应用的核心需求。从在线聊天系统到实时股票行情，再到多人协作编辑工具，这些场景都依赖于高效的实时数据传输。传统的<span
lang=EN-US> HTTP </span>协议由于其请求<span lang=EN-US>-</span>响应模式的限制，并不适合这类应用。今天我将分享我学习的<span
lang=EN-US> WebSocket </span>协议，它是如何实现真正的全双工通信的。<span lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>WebSocket </span>握手机制<span lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>WebSocket </span>连接的建立始于一个特殊的<span
lang=EN-US> HTTP </span>握手过程：<span lang=EN-US><o:p></o:p></span></p>

<ol style='margin-top:0cm' start=1 type=1>
 <li class=MsoNormal style='mso-list:l1 level1 lfo1;tab-stops:list 36.0pt'>客户端发送<span
     lang=EN-US> HTTP GET </span>请求，携带以下关键头信息：<span lang=EN-US><o:p></o:p></span></li>
</ol>

<p class=MsoNormal><span lang=EN-US>Upgrade: websocket<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Connection: Upgrade<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Sec-WebSocket-Key: [</span>随机生成的<span
lang=EN-US>base64</span>编码字符串<span lang=EN-US>]<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Sec-WebSocket-Version: 13<o:p></o:p></span></p>

<ol style='margin-top:0cm' start=2 type=1>
 <li class=MsoNormal style='mso-list:l1 level1 lfo1;tab-stops:list 36.0pt'>服务器验证这些头信息后，返回<span
     lang=EN-US> 101 </span>状态码（<span lang=EN-US>Switching Protocols</span>）响应：<span
     lang=EN-US><o:p></o:p></span></li>
</ol>

<p class=MsoNormal><span lang=EN-US>HTTP/1.1 101 Switching Protocols<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Upgrade: websocket<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Connection: Upgrade<o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>Sec-WebSocket-Accept: [</span>基于客户端<span
lang=EN-US>Key</span>计算出的验证值<span lang=EN-US>]<o:p></o:p></span></p>

<p class=MsoNormal>这个握手过程完成了从<span lang=EN-US> HTTP </span>到<span lang=EN-US>
WebSocket </span>协议的切换，之后通信将基于<span lang=EN-US> WebSocket </span>协议进行。<span
lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal>全双工通信模式<span lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>WebSocket </span>最引人注目的特性是其全双工通信能力：<span
lang=EN-US><o:p></o:p></span></p>

<ul style='margin-top:0cm' type=disc>
 <li class=MsoNormal style='mso-list:l0 level1 lfo2;tab-stops:list 36.0pt'>双向同时传输：客户端和服务器可以同时发送消息，无需等待对方完成<span
     lang=EN-US><o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l0 level1 lfo2;tab-stops:list 36.0pt'>持久连接：一旦建立连接，可以长期保持，避免重复握手开销<span
     lang=EN-US><o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l0 level1 lfo2;tab-stops:list 36.0pt'>低延迟：消息到达后立即被处理，无需等待新的<span
     lang=EN-US> HTTP </span>请求<span lang=EN-US><o:p></o:p></span></li>
</ul>

<p class=MsoNormal>这种模式特别适合需要实时交互的应用场景，如在线游戏、协同编辑、实时监控等。<span lang=EN-US><o:p></o:p></span></p>

<p class=MsoNormal><span lang=EN-US>WebSocket </span>与<span lang=EN-US> HTTP </span>的核心差异<span
lang=EN-US><o:p></o:p></span></p>


特性 | HTTP | WebSocket
-- | -- | --
连接性质 | 短连接，每次请求后关闭 | 长连接，建立后可长期复用
通信方向 | 单向（客户端请求，服务器响应） | 全双工（双方可随时发送消息）
数据格式 | 包含大量头部的完整报文 | 轻量帧结构（头部仅2-10字节）
适用场景 | 传统网页浏览 | 实时应用



<p class=MsoNormal><span lang=EN-US><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span lang=EN-US>WebSocket </span>的优势<span lang=EN-US><o:p></o:p></span></p>

<ol style='margin-top:0cm' start=1 type=1>
 <li class=MsoNormal style='mso-list:l3 level1 lfo3;tab-stops:list 36.0pt'>高效传输：相比<span
     lang=EN-US> HTTP</span>，<span lang=EN-US>WebSocket </span>的头部开销极小，特别适合高频小数据量传输<span
     lang=EN-US><o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l3 level1 lfo3;tab-stops:list 36.0pt'>服务器推送：服务器可以主动向客户端推送数据，无需客户端轮询<span
     lang=EN-US><o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l3 level1 lfo3;tab-stops:list 36.0pt'>减少延迟：避免了<span
     lang=EN-US> HTTP </span>的重复连接建立过程<span lang=EN-US><o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l3 level1 lfo3;tab-stops:list 36.0pt'>节省带宽：长连接避免了每次通信的<span
     lang=EN-US> TCP/IP </span>握手和<span lang=EN-US> TLS </span>协商开销<span
     lang=EN-US><o:p></o:p></span></li>
</ol>
